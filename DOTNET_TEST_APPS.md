# .NET Test Applications for C# Analyzer Rules

This document lists recommended .NET applications for testing the C# analyzer rules generated by this tool.

## Recommended Test Applications

### 1. Microsoft eShopModernizing (Highly Recommended)

**Repository:** [dotnet-architecture/eShopModernizing](https://github.com/dotnet-architecture/eShopModernizing)

**Description:** Official Microsoft sample for modernizing legacy .NET Framework applications to .NET Core.

**Why Use This:**
- Official Microsoft sample code
- Specifically designed for modernization testing
- Contains two legacy ASP.NET 4.x applications:
  - `eShopLegacyMVCSolution` (ASP.NET 4.x MVC)
  - `eShopLegacyWebFormsSolution` (ASP.NET 4.x Web Forms)
- Well-documented migration path
- Most likely to contain the patterns our rules detect (System.Web.Http, System.Web.Mvc, etc.)

**Clone:**
```bash
git clone https://github.com/dotnet-architecture/eShopModernizing.git
cd eShopModernizing
```

**Documentation:**
- [Tour of the ASP.NET MVC and WebForms apps](https://github.com/dotnet-architecture/eShopModernizing/wiki/01.-Tour-of-the-ASP.NET-MVC-and-WebForms-apps-implementation-code)

## Testing with Konveyor Analyzer

Once you've cloned a test application, you can run the C# analyzer rules against it using Konveyor's analyzer-lsp.

### Prerequisites

1. Install Konveyor analyzer-lsp
2. Ensure the C# provider is available

### Running Analysis

```bash
# Navigate to the test application directory
cd /path/to/test-app

# Run analyzer with your generated rules
kantra analyze \
  --input /path/to/test-app \
  --output ./analysis-output \
  --rules /path/to/analyzer-rule-generator/examples/output/dotnet-test/
```

### Expected Patterns to Detect

Our generated rules should detect patterns like:

- **System.Web.Http** namespace usage (ASP.NET Web API → ASP.NET Core)
- **System.Web.Mvc** namespace usage (ASP.NET MVC → ASP.NET Core MVC)
- **Entity Framework 6** patterns (EF6 → EF Core)
- Removed/deprecated APIs in .NET Core
- Configuration changes (web.config → appsettings.json)

## Recommendation

**Start with eShopModernizing** for the most comprehensive testing:

```bash
# Clone the repository
git clone https://github.com/dotnet-architecture/eShopModernizing.git
cd eShopModernizing

# Test against the MVC solution
kantra analyze \
  --input ./src/eShopLegacyMVCSolution \
  --output ./analysis-mvc-output \
  --rules /path/to/analyzer-rule-generator/examples/output/dotnet-test/

# Test against the WebForms solution
kantra analyze \
  --input ./src/eShopLegacyWebFormsSolution \
  --output ./analysis-webforms-output \
  --rules /path/to/analyzer-rule-generator/examples/output/dotnet-test/
```

This will give you the best coverage of real-world .NET Framework patterns that need migration to .NET Core.
